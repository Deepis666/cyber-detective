### ✅ 2. 状态管理双层架构（gameState.js 重写）

将单案件状态升级为**跨案件层 + 单案件层**两层架构：

```
跨案件层（持久化，不随案件切换重置）：
  - gamePhase: 'menu'|'briefing'|'investigate'|'interrogation'|'hub'|'ending'|'final_ending'
  - currentCaseIndex: 0/1/2/3
  - casesResults: [{ caseId, ending, score, keyFlags, completedAt }]
  - unlockedHiddenCase: boolean
  - globalFlags: {}

单案件层（切换案件时重置）：
  - currentPhase / currentScene / currentSuspect
  - evidenceObtained / evidenceExamined / dialogueHistory
  - stressLevel / score / endingTriggered / flags
```

新增方法：
- `startNewCase(caseIndex)` — 重置单案件层，保留跨案件层
- `recordCaseResult(result)` — 记录案件结果到 casesResults
- `checkHiddenCaseUnlock()` — 检查前3案是否全好结局，解锁隐藏案件
- `setGamePhase()` / `getGamePhase()` — 跨案件阶段管理
- `getCasesResults()` / `getCaseResult()` / `getCurrentCaseIndex()`
- `initSuspectStress(suspectIds)` — 按案件动态初始化压力值
- `getFinalEndingType()` — 计算最终结局类型（all_good/mixed/all_bad）
- `isAllMainCasesCompleted()` — 判断主线全通
- `setGlobalFlag()` / `getGlobalFlag()` — 跨案件标记

存档兼容：`loadGame()` 自动补充旧版存档缺失的跨案件层字段，version=2。

### 案件流转状态机

```
新游戏 → startCase(0) → 简报 → 调查 → 审讯 → 结案
                                                ↓
                                          ending画面 → back-hub
                                                          ↓
                                                       Hub画面 → next-case
                                                          ↓
                                              getNextCaseIndex() 决定:
                                                ├─ 有未完成案件 → startCase(next)
                                                └─ 全部完成 → renderFinalEnding()
                                                              ↓
                                                         final_ending画面 → back-menu

隐藏案件解锁：案件0/1/2 全 ending_good → checkHiddenCaseUnlock() → unlockedHiddenCase=true
            → getNextCaseIndex() 返回 3 → startCase(3)
```
