# 腾讯云黑客松 · 赛博朋克侦探叙事游戏 — 完整计划

## 一、对现有创意的建议

### ✅ 方案核心优势
1. **类型契合度高**：赛博朋克 + 逆转裁判 + AI 叙事，完美切中「赛题三：AI 重塑叙事体验」
2. **技术可行性高**：单案件 + 3 结局 + 简化架构，5 天可落地
3. **演示效果强**：AI 动态对话天然适合录屏展示，容易给评审留下印象
4. **工具链闭环**：CodeBuddy 生成内容 → WorkBuddy 生成美术 → CloudStudio 部署，链路完整

### ⚡ 建议优化点

#### 1. 技术栈建议：纯前端单页应用（HTML/CSS/JS）
- **理由**：5 天极限开发，浏览器可玩链接是硬要求，纯前端部署最简单
- **框架**：Vanilla JS + Vite（构建快、零配置）
- **AI 调用**：前端直接 fetch 混元 API，无需后端中转（注意前端暴露 API Key 问题，建议用 CloudStudio 环境变量或比赛期间临时方案）
- **数据持久**：LocalStorage 存游戏进度和对话历史，足够 Demo 需求

#### 2. 玩法建议：聚焦「一个高光时刻」
- 现有方案亮点多（动态证据 + 心理系统 + 证据组合），但 5 天建议**主攻一个做到极致**
- **推荐保留**：「证据组合推理」+「嫌疑人心理状态变化」—— 这两个互动感最强，录屏最有冲击力
- **建议弱化**：「AI 动态生成调查文本」改为预设文本 + AI 润色，减少调用延迟和不稳定性

#### 3. 叙事结构建议：单一场景深挖
- 现有：1 个案件，3 个调查场景 + 1 次审讯
- **优化**：1 个案件 = 1 个核心场景（如：一间赛博朋克公寓犯罪现场）+ 1 次审讯高潮
- **好处**：场景美术只需 1 张高质量背景，角色立绘集中 2-3 个嫌疑人，美术工作量减半
- **叙事**：通过时间回溯/线索重组来替代多场景切换，同样能做出层次感

#### 4. AI Prompt 架构建议
- 采用 **JSON Schema 约束** 替代自由文本输出，确保前端好解析
- 案件事实基底用硬编码 JSON，AI 只负责「回应文本生成」+「情绪状态变化」
- 预设 3 套「压力等级」回应模板（冷静→紧张→崩溃），AI 在模板间选择和润色，而非完全自由生成，大幅降低幻觉风险

#### 5. 录屏/Demo 建议
- 评审不会玩完整个游戏，**前 3 分钟体验决定一切**
- 开场 30 秒要有强氛围（赛博朋克 BGM + 霓虹 UI + 案件简报动画）
- 录屏重点展示：「出示证据 → AI 嫌疑人情绪变化 → 新对话分支」这一循环

## 三、三人小组并行工期安排

### 推荐分工（按专业匹配）

| 专业 | 角色 | 代号 | 核心职责 | 专业优势 |
|------|------|------|----------|----------|
| **软件工程** | **程序主程** | A | 游戏框架搭建、所有系统开发、部署、版本管理 | 工程能力强，负责代码架构、模块拆分、CloudStudio 部署 |
| **人工智能** | **AI 架构师** | B | 混元 API 接入、Prompt 工程、嫌疑人心理系统、证据组合推理逻辑 | 懂模型原理，能设计 Prompt 模板、情绪状态机、离线降级方案 |
| **城乡规划** | **美术叙事导演** | C | 世界观文案、空间场景设计、AI 生图/生声音、PPT、视频剪辑 | 有空间美学和汇报经验，负责视觉风格定调、叙事节奏、后期制作 |

### 分工细化说明

- **A（软件工程）**：不写 AI Prompt，只负责把 B 设计好的 Prompt 和 API 接口封装进游戏代码；专注前端工程化（Vite 配置、组件化、状态管理）。
- **B（人工智能）**：不碰 UI 样式代码，专注「AI 系统」—— Prompt 模板设计、JSON Schema 约束、情绪状态机逻辑、API 降级策略；同时是 AI 效果的「质检员」。
- **C（城乡规划）**：不碰核心代码，专注「内容生产」和「包装输出」—— 用 WorkBuddy/Miora 批量生成立绘/背景/BGM、撰写案件剧本、设计 PPT 和视频分镜。

## 四、风险与应急预案

| 风险 | 应对方案 |
|------|----------|
| AI API 调用延迟高/失败 | 准备「离线模式」：预设 20 组高质量回应，API 失败时自动切换 |
| AI 生成内容幻觉严重 | 案件事实基底硬编码，AI 只负责「润色表达」而非「创造事实」 |
| 美术生成质量不稳定 | 提前准备 2-3 套风格参考图，用 img2img 固定风格 |
| Day 3 核心功能未跑通 | 启动「降级方案」：取消 AI 实时接入，改用预设对话树 + 有限分支 |
| 部署失败 | 本地录屏仍可提交，同时准备 GitHub Pages 作为备用部署 |
