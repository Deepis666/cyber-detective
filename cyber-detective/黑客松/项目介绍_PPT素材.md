## 项目概述

**一句话简介**  
一款以 DeepSeek 大模型驱动叙事的赛博朋克侦探推理游戏。玩家扮演侦探，通过场景调查、证据推理、AI 实时审讯层层推进，最终揭露真相。

**核心数据**  
- 代码行数：~3000+ 行  
- 模块数：8 个核心模块  
- 案件数量：4 个完整主线案件（含 1 个隐藏案件）  
- AI 模型：DeepSeek Chat（V3）  
- 技术栈：Vanilla JS + Vite + CSS 纯前端  

---

## 游戏特色

**1. AI 驱动的动态叙事**  
- 不再依赖固定对话树，嫌疑人的每句回应由 DeepSeek 实时生成  
- 根据压力值、对话历史、已获证据动态调整角色情绪与回答内容  
- 支持"提问/施压/出示证据"三种审讯动作，AI 即兴反馈

**2. 证据链推理系统**  
- 现场收集线索，自由组合两件证据，AI 分析逻辑关联  
- 四条推理维度：物理因果链、人物指向链、时间逻辑链、涌现发现  
- 组合成功解锁新证据，形成完整的推理闭环

**3. 多案件主线叙事**  
- 4 个独立案件通过"事务所 Hub"串联成完整故事  
- 前 3 个案件全部达成好结局 → 解锁第 4 个隐藏案件  
- 结局由 AI 根据玩家评分和关键选择动态生成

**4. 赛博朋克美学**  
- 霓虹光效 CSS 主题（青蓝/品红/紫色调）  
- 打字机风格对话系统  
- 高压审讯心理面板（冷靜/紧张/崩溃三态）  
- 全沉浸式 HUD 界面

## AI 集成方案

### 覆盖的 AI 场景（5 个 Prompt）

| 场景 | Prompt 行数 | 返回数据 | 离线回退 |
|------|------------|---------|---------|
| 审讯对话 | 98 行 | 回应文本、情绪、压力变化、新线索 | 预设回应池（每个嫌疑人 3 种状态×2-4 条） |
| 证据组合 | 63 行 | 是否相关、推理文本、解锁证据 ID | 案件预设 3 对+通用无关回复 |
| 场景调查 | 84 行 | 场景描述、证据发现、氛围句 | 通用场景描述 |
| 结局生成 | 78 行 | 结局叙事 200-400 字、新闻简报 50-80 字 | 硬编码结局文本 |
| 剧情旁白 | 26 行 | 局势分析、下一步建议 | 简单提示文本 |

### AI 配置参数

- **模型**：DeepSeek Chat（deepseek-chat）
- **API 格式**：OpenAI 兼容接口
- **温度**：0.7
- **Max Tokens**：500
- **超时**：15 秒
- **上下文**：最近 10 条对话
- **重试策略**：指数退避（最多 2 次）
- **输出格式**：JSON Object

### 接入流程

```
玩家操作 → 加载 Prompt 模板 (变量替换) → showAIThinking()
→ callAPI (fetch + 超时 + 重试) → JSON 解析 → 数据校验
→ 失败? → 指数退避 → 全部失败? → 离线预设回退
→ 成功? → 应用到游戏状态 (评分/压力/证据)
```

## 游戏平衡体系

### 评分系统（设计目标：好结局需要 120 分）

| 操作 | 奖励 | 说明 |
|------|------|------|
| 戳穿谎言 | +12 | 用证据质问嫌疑人 |
| 出示相关证据 | +10 | 审讯时出示关键证物 |
| AI 发现矛盾 | +7 | 审问中自动揭露矛盾 |
| 证据组合（相关） | +7 | 两件证据逻辑关联成功 |
| 出示无关证据 | -3 | 浪费机会扣分 |
| 草草结案 | -5/条 | 每漏一条线索扣分 |

### 压力系统

| 压力范围 | 情绪 | 效果 |
|---------|------|------|
| 0 - 29 | 冷静 | 滴水不漏，防御高 |
| 30 - 69 | 紧张 | 露出破绽，透露信息 |
| 70 - 100 | 崩溃 | 全盘托出，交代真相 |

### 结局分值

| 结局 | 条件 | 风格 |
|------|------|------|
| 好结局 | 评分 ≥ 120 | 真相大白，灰色黎明 |
| 普通结局 | 60 - 119 | 未完成遗憾，黄昏霓虹 |
| 坏结局 | < 60 | 错判冤案，深夜暴雨 |

## 运行方式

```bash
# 1. 安装依赖
npm install

# 2. 配置 API Key（DeepSeek）
复制 .env.example 为 .env，填入：
VITE_HUNYUAN_API_KEY=sk-xxx
VITE_HUNYUAN_API_URL=https://api.deepseek.com/v1/chat/completions

# 3. 开发运行
npm run dev

# 4. 生产构建
npm run build
```

### 离线模式说明

即使不配置 API Key，游戏依然可以完整运行（使用预设对话池回退），适合展示和演示。

## Q&A 备选

**Q: 为什么不做成 Unity 或 Unreal？**  
A: 纯 Web 前端 + 大模型 API 是黑客松的最优解——无需下载安装、零部署成本、AI 调用直接嵌入浏览器。

**Q: 游戏时长短吗？**  
A: 4 个案件各约 20-30 分钟，总计约 1.5-2 小时沉浸体验。

**Q: 未来规划？**  
A: 加入案件编辑器让玩家自己写故事；Steam 独立发布计划；多语言支持。
