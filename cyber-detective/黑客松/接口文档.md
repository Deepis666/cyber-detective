## 一、项目架构概览

```

│   ├── evidence.json       # 证据定义

├── modules/                # 核心模块
│   ├── gameState.js        # 全局状态管理（A 实现）
│   ├── dialogueSystem.js   # 对话系统（A 实现）
│   ├── evidenceSystem.js   # 证据系统（A 实现）
│   ├── aiEngine.js         # AI 接入引擎（A 封装，B 提供 Prompt）
│   ├── sceneManager.js     # 场景管理（A 实现）
│   └── audioManager.js     # 音频管理（A 实现）
└── prompts/                # AI Prompt 模板（B 编写，A 调用）
    ├── interrogation.txt
    ├── evidenceCombine.txt
    └── sceneInvestigate.txt
```

### 技术栈

- **构建工具**：Vite（零配置，热更新快）
- **前端**：Vanilla JS（ES6 Modules），无框架
- **样式**：CSS3 + CSS Variables（霓虹主题配色）
- **AI 调用**：前端直接 fetch 混元 API（HTTPS）
- **数据持久化**：LocalStorage
- **部署**：CloudStudio 静态托管 / EdgeOne Pages

## 三、模块接口定义

### 3.1 全局状态管理 `gameState.js`

**由 A 实现，各模块调用。**

```javascript
// ====================
// State Structure
// ====================
const defaultState = {
  currentPhase: 'intro',      // 'intro' | 'investigate' | 'interrogation' | 'ending'
  currentScene: null,         // 当前调查场景 ID
  currentSuspect: null,       // 当前审讯嫌疑人 ID
  evidenceObtained: [],       // 已获得证据 ID 列表
  evidenceExamined: [],       // 已查看证据 ID 列表
  dialogueHistory: [],        // 对话历史（用于 AI 上下文）
  stressLevel: {              // 各嫌疑人压力值 0-100
    suspect_001: 0,
    suspect_002: 0
  },
  score: 0,                   // 推理评分
  endingTriggered: null,      // 触发的结局 ID
  flags: {}                   // 剧情标记（如 "found_key_clue": true）
};

// ====================
// Public API
// ====================

/**
 * 获取当前完整状态（深拷贝）
 * @returns {Object} state
 */
function getState();

/**
 * 更新状态（支持局部合并）
 * @param {Object} patch - 要合并的状态片段
 * @returns {Object} 更新后的完整状态
 */
function updateState(patch);

/**
 * 添加证据
 * @param {string} evidenceId
 */
function addEvidence(evidenceId);

/**
 * 增加嫌疑人压力值
 * @param {string} suspectId
 * @param {number} delta - 增量（可负）
 * @returns {number} 更新后的压力值
 */
function addStress(suspectId, delta);

/**
 * 获取嫌疑人当前情绪标签
 * @param {string} suspectId
 * @returns {string} 'calm' | 'nervous' | 'broken'
 */
function getEmotion(suspectId);

/**
 * 推进评分
 * @param {number} delta
 */
function addScore(delta);

/**
 * 设置剧情标记
 * @param {string} flag
 * @param {any} value
 */
function setFlag(flag, value);

/**
 * 存档到 LocalStorage
 */
function saveGame();

/**
 * 从 LocalStorage 读档
 * @returns {Object|null}
 */
function loadGame();
```

**由 A 封装接口，B 提供 Prompt 模板。**

```javascript

  apiUrl: 'https://hunyuan.tencentcloudapi.com/...', // 混元 API 端点
```

## 四、AI Prompt 接口规范

### 4.1 Prompt 模板管理

所有 Prompt 模板存放在 `prompts/` 目录下，**由 B 编写，A 负责在代码中读取和变量替换。**

```javascript
// aiEngine.js 中读取模板示例
async function loadPrompt(templateName, variables) {
  const template = await fetch(`/prompts/${templateName}.txt`).then(r => r.text());
  return template.replace(/\{\{(\w+)\}\}/g, (match, key) => variables[key] || match);
}
```

### 4.2 审讯 Prompt `prompts/interrogation.txt`

```text
你是一名赛博朋克侦探游戏中的嫌疑人角色。请严格按照以下设定回应。

【案件事实基底】（不可改变）
{{caseTruth}}

【你的角色设定】
姓名：{{suspectName}}
身份：{{suspectIdentity}}
性格：{{suspectPersonality}}
你知道的真相：{{suspectKnowledge}}
你的谎言：{{suspectLies}}

【当前状态】
压力值：{{currentStress}}/100
情绪：{{currentEmotion}}

【对话历史】
{{dialogueHistory}}

【玩家行动】
类型：{{actionType}}
内容：{{playerContent}}

【要求】
1. 以第一人称回应，保持角色一致性
2. 如果玩家出示了戳穿你谎言的证据，压力必须上升，情绪恶化
3. 如果玩家出示无关证据，你可以不屑或困惑
4. 输出必须是 JSON 格式：
{
  "response": "你的回应文本（50-150字）",
  "emotion": "calm|nervous|broken",
  "stressDelta": 数字（-20到+30）,
  "hasContradiction": true|false,
  "newClue": "null 或新线索文本"
}
```

### 4.4 API 请求/响应格式

```javascript
// ===== 请求格式（混元 API）=====
const requestBody = {
  model: "hunyuan-lite",
  messages: [
    { role: "system", content: promptTemplate },
    ...history
  ],
  temperature: 0.7,
  max_tokens: 500,
  response_format: { type: "json_object" } // 若支持
};

// ===== 期望响应 =====
const response = {
  choices: [{
    message: {
      content: '{"response":"...","emotion":"nervous","stressDelta":15,...}'
    }
  }]
};

// ===== A 的封装层负责 =====
// 1. 解析 JSON（失败时 fallback）
// 2. 校验字段完整性
// 3. 超时处理（8秒）
// 4. 重试机制（最多2次）
```

## 七、关键对接约定

### A（软件工程）↔ B（AI 架构师）

| 对接点 | A 的职责 | B 的职责 |
|--------|----------|----------|
| Prompt 模板 | 读取模板文件，进行变量替换和 API 调用 | 编写 `prompts/*.txt`，定义变量名和 JSON 输出格式 |
| API 响应解析 | 封装 fetch，解析 JSON，错误 fallback | 定义 JSON 字段规范，提供测试用例 |
| 状态机 | 实现压力值计算和情绪映射 | 设计压力-情绪转换规则，校准参数 |
| 离线模式 | 实现 fallbackResponse() 函数 | 提供 20 组高质量预设回应数据 |

### A（软件工程）↔ C（美术叙事）

| 对接点 | A 的职责 | C 的职责 |
|--------|----------|----------|
| 数据文件 | 读取 `data/*.json`，校验格式 | 按本文档格式填充内容 |
| 资源路径 | 按约定路径加载图片/音频 | 按 `assets/` 目录结构存放资源 |
| UI 样式 | 实现 CSS 变量和布局 | 提供配色方案、字体选择参考 |
| 文本长度 | 对话框支持动态高度 | 单条对话控制在 150 字以内 |

### B（AI 架构师）↔ C（美术叙事）

| 对接点 | B 的职责 | C 的职责 |
|--------|----------|----------|
| 案件事实 | 设计 JSON 结构，定义不可变事实 | 撰写世界观、角色背景、案件故事 |
